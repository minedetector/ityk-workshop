Key Features

  • Instant Synchronization: HTML/CSS changes sync to container in milliseconds
  • No Full Rebuilds: Skip expensive image rebuilds for static file changes
  • Native Docker: Uses Docker Compose Watch - no external tools needed
  • Team Productivity: Every developer can test changes immediately
  • Scalable Pattern: Same approach works for all microservices

Running the Demo

  1. Clone or download this project to your machine
  2. Navigate to the project directory
  3. Run: docker compose up --build
  4. In another terminal, run: docker compose watch
  5. Edit the HTML files and see changes immediately!

What Happens When You Edit Files

HTML/CSS files in ./html/:

  • Changes are synced to /usr/share/nginx/html in the container
  • Nginx serves updated files immediately (no restart needed)
  • Browser refresh shows your changes in seconds

Dockerfile or dependencies:

  • Container is rebuilt automatically
  • Quick rebuild due to Docker layer caching

Best Practices for Your Team

  • Document the workflow: Keep your docker-compose.yml in version control with the watch section
  • Consistent paths: Establish naming conventions for watch targets across all services
  • Use .dockerignore: Exclude node_modules, vendor, etc. from rebuilds
  • Layer caching: Order Dockerfile instructions to maximize cache hits
  • Team onboarding: Document commands: docker compose up --build + docker compose watch
